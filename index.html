Что такое Figma и зачем она тебе основа🔗
Figma — это онлайн-редактор для создания дизайна интерфейсов. Работает прямо в браузере, бесплатно. Дизайнеры делают макеты, фронтендеры по ним верстают.
Зачем это фронтендеру?
Твой менеджер или дизайнер пришлёт тебе ссылку на Figma — и ты должен по ней сделать точно такой же сайт. Figma покажет все размеры, цвета, отступы и шрифты — прямо в пикселях. Тебе не нужно угадывать — просто смотри и копируй.
1
Дизайнер делает макет в Figma
Рисует кнопки, карточки, шапки, цвета, шрифты — всё визуальное оформление сайта.
2
Делится ссылкой с тобой
Ты открываешь файл и видишь точно тот же дизайн. Можно смотреть, но не редактировать.
3
Ты кликаешь на элемент → видишь CSS
В правой панели Figma показывает: размер, отступы, цвет, шрифт, border-radius и даже готовый CSS-код.
4
Копируешь стили — вставляешь в код
Нажимаешь «Copy as CSS» и получаешь готовые стили. Осталось только вставить в свой файл.
Интерфейс Figma — что где находится основа🔗
Figma состоит из четырёх зон. Запомни их — и ты перестанешь теряться.

Слева. Список всех элементов: фреймы, группы, тексты, картинки.

Экспорт картинок и иконок важно🔗
1

2

Нажми «+» в секции Export. Выбери формат: PNG (фото), SVG (иконки), WebP (оптимизация).

Какой формат выбрать:

SVG — иконки и логотипы. Масштабируются без потери качества.

Компоненты, стили и переменные PRO🔗
Компоненты — это как шаблоны. Дизайнер создаёт кнопку один раз, а потом использует её везде. Для тебя это сигнал: нужен переиспользуемый HTML + CSS блок.

Стили (Styles) — это цвета, шрифты и тени, вынесенные в переменные. В CSS это будут :root { --color-primary: ... }.

Variables — новая фича Figma. Одна переменная меняет цвет сразу везде. Это как CSS-переменные в дизайне. Когда дизайнер использует переменные — ты тоже должен использовать var(--...) в CSS.
💡 Увидел фиолетовый значок ◆ рядом со слоем? Это компонент. Посмотри как он называется и создай отдельный CSS-класс с таким же именем.
Алгоритм верстки по макету Figma практика🔗
Когда ты получаешь ссылку на Figma, действуй по этому плану — и никогда не потеряешься.
1
Изучи макет целиком
Открой Figma. Посмотри на всю страницу — что тут вообще? Шапка, секции, подвал. Поняли структуру — идём дальше.
2
Посмотри сколько breakpoints
Есть ли мобильная версия? Tablet? Их должно быть несколько фреймов рядом. Запомни ширины.
3
Собери цвета и шрифты
Открой Inspect любого текста — запиши font-family, размеры, цвета. Занеси в :root как CSS-переменные.
4
Верстай сверху вниз, снаружи внутрь
Сначала header, потом секции, потом footer. В каждой секции — сначала контейнер, потом его содержимое.
5
Проверяй каждый элемент через Inspect
Кликаешь в Figma — смотришь CSS справа — пишешь в свой файл. Не угадывай размеры!
6
Pixel perfect? Сравни наложением
Открой оба в соседних вкладках. Или используй плагин PixelPerfect — он накладывает макет поверх страницы.
Design Tokens: цвета, отступы, шрифты система🔗
Design Tokens — это общие переменные дизайна. В Figma они называются Variables/Styles, в CSS — custom properties.
/* CSS-переменные из макета Figma */ :root { --color-primary: #7c3aed; --color-bg: #0f172a; --color-text: #e5e7eb; --font-main: Inter, system-ui, sans-serif; --text-sm: 14px; --text-base: 16px; --text-xl: 24px; --space-2: 8px; --space-3: 12px; --space-4: 16px; --radius-sm: 8px; --radius-md: 12px; } .button { background: var(--color-primary); padding: var(--space-3) var(--space-4); border-radius: var(--radius-sm); }
Зачем токены: если главный цвет поменялся в макете, ты меняешь одну переменную, а не 40 классов.
Что выносить: основные цвета, размеры текста, отступы, скругления, тени, ширину контейнера.
Как работать: сначала собери токены из Figma, потом верстай компоненты через var(--...).

🟢 Node.js — JavaScript на сервере

Раньше JavaScript жил только в браузере. Node.js позволяет запускать JS на сервере — и теперь одним языком можно писать и фронтенд, и бэкенд. Это твой следующий шаг после того как уверенно освоишь JS. Node.js — основа для серверов, API, чатов, и сайтов которые «выдерживают много людей».

⚠️ Не начинай Node.js пока не уверен в обычном JS. Node — это тот же JavaScript, но на сервере. Сначала научись писать функции, циклы, работать с массивами и объектами в браузере. Потом Node пойдёт легко.

🗄️ SQL — базы данных

localStorage хранит данные только в одном браузере. Чтобы данные жили по-настоящему — нужна база данных. SQL — это язык запросов к базам. Освоишь его — и сможешь хранить пользователей, посты, заказы, что угодно. Это обязательная часть бэкенда.

🖥️ Серверы, кэш, балансировка, Docker

Это то что делает сайт способным «выдерживать много людей». Здесь нет программирования в обычном смысле — это инженерия систем: как устроены серверы, как ускорять, как масштабировать. Тема большая и продвинутая — это не для новичка, но полезно знать куда расти.

⚠️ Это далёкая цель, не сейчас. Сначала фронтенд → потом Node.js → потом базы данных → и только потом эта тема. Здесь даётся обзорно, чтобы ты понимал общую картину и куда ведёт путь.

grid-area: header
Поместить в именованную зону (из grid-template-areas).

Выравнивание конкретного элемента внутри ячейки.
Добавление и удаление
push()
arr.push(4, 5)
Добавляет в конец. Возвращает новую длину.
pop()
arr.pop()
Удаляет и возвращает последний элемент.
unshift()
arr.unshift(0)
Добавляет в начало. Возвращает новую длину.
shift()
arr.shift()
Удаляет и возвращает первый элемент.
splice()
arr.splice(2, 1) | arr.splice(1, 0, 'x')
Удалить / вставить элементы по индексу. Мутирует массив.
Трансформация (возвращает новый массив)
map()
arr.map(n => n * 2)
Преобразует каждый элемент. Новый массив той же длины.
filter()
arr.filter(n => n > 3)
Оставляет только те, где callback вернул true.
reduce()
arr.reduce((sum, n) => sum + n, 0)
Сворачивает массив в одно значение. Второй аргумент — начальное.
flat()
arr.flat() | arr.flat(Infinity)
Разворачивает вложенные массивы. Аргумент — глубина.
flatMap()
arr.flatMap(x => [x, x*2])
map() + flat(1). Очень удобно для создания пар.
slice()
arr.slice(1, 3) | arr.slice(-2)
Срез массива. Не мутирует. Отрицательный = с конца.
concat()
arr.concat([4, 5]) | [...a, ...b]
Объединить массивы. Не мутирует.
sort()
arr.sort((a, b) => a - b)
Сортировка на месте (мутирует!). Без callback — лексикографически.
reverse()
arr.reverse()
Разворачивает массив на месте (мутирует).
Поиск и проверка
find()
arr.find(x => x.id === 3)
Первый элемент, где callback = true. Или undefined.
findIndex()
arr.findIndex(x => x > 5)
Индекс первого найденного. Или -1.
includes()
arr.includes(3)
Есть ли значение в массиве. true / false.
indexOf()
arr.indexOf(3)
Индекс первого вхождения. Или -1.
some()
arr.some(n => n > 10)
Хотя бы один элемент прошёл условие? true / false.
every()
arr.every(n => n > 0)
Все элементы прошли условие? true / false.
forEach()
arr.forEach((el, i) => { ... })
Перебрать массив. Не возвращает ничего (undefined).
join()
arr.join(', ') | arr.join('')
Объединить элементы в строку через разделитель.
Array.from()
Array.from('abc') | Array.from({length:3},(_,i)=>i)
Создать массив из итерируемого или объекта.
Array.isArray()
Array.isArray(val)
Проверить, является ли значение массивом.
Информация о строке
.length
str.length
Длина строки в символах.
at() / charAt()
str.at(-1) | str.charAt(0)
Символ по индексу. at() поддерживает отрицательные.
indexOf()
str.indexOf('мир') | lastIndexOf()
Позиция подстроки. -1 если не найдено.
includes()
str.includes('привет')
Есть ли подстрока. true / false.
startsWith / endsWith
str.startsWith('http') | str.endsWith('.js')
Начинается / заканчивается ли строка подстрокой.
Изменение строки
toUpperCase / toLowerCase
str.toUpperCase() | str.toLowerCase()
Верхний / нижний регистр.
trim()
str.trim() | trimStart() | trimEnd()
Убрать пробелы по краям.
replace() / replaceAll()
str.replace('a','b') | replaceAll('a','b')
Заменить первое / все вхождения.
slice()
str.slice(0, 5) | str.slice(-3)
Вырезать подстроку. Поддерживает отрицательные индексы.
substring()
str.substring(1, 4)
Вырезать подстроку. Не поддерживает отрицательные.
split()
str.split(',') | str.split('')
Разбить строку в массив по разделителю.
padStart / padEnd
str.padStart(5, '0') | padEnd(10, '.')
Дополнить строку символами до нужной длины.
repeat()
str.repeat(3)
Повторить строку N раз.
match() / matchAll()
str.match(/\d+/g)
Найти по RegExp. matchAll — итератор всех совпадений.
